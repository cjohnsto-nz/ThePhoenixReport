- Content blocks: 5 x 10 minutes
- Closing / discussion buffer: 5 minutes

## Storytelling rules for delivery

- Use the card reveal to introduce the beat.
- Use the global step to explain why that beat changes our understanding of the system.
- Revisit earlier ideas in the narration rather than re-revealing the same item.
- Keep transitions explicit: problem -> visibility -> flow -> feedback -> learning.

## Visual recommendations by block

### Block 1

- Payroll failure cascade timeline
- Brent as visible constraint overlay on the org chart
- Invisible work board: questions Bill asks that nobody can answer
- Audit severity pyramid or control deficiency stack

### Block 2

- WIP stack versus completion rate
- Brent bottleneck funnel
- Change completion chart: work submitted versus work finished
- Four Types of Work quadrant or pie breakdown

### Block 3

- Unplanned work displacing planned work
- Queue build-up at the constraint
- Project freeze before/after flow comparison
- First Way diagram: optimize the whole system

### Block 4

- Time to task attention versus capacity curve
- Handoff swimlane showing waiting time versus touch time
- Deployment failure loop: big batch -> late discovery -> expensive fix
- Business risk mapping slide tying IT issues to commercial outcomes

### Block 5

- Unicorn versus old delivery model comparison
- Deployment pipeline diagram
- Chaos / resilience experiment timeline for Narwhal
- External constraint map: vendor or outsourced platform as the new bottleneck

## Quote guidance

- Prefer very short excerpts or paraphrases.
- Use quotes to punctuate a slide, not carry the explanation.
- Favor lines that reveal stakes, not lines that require book context.
